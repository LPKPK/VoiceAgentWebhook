# Retell AI Webhook Test

## Overview

This project demonstrates how to integrate a FastAPI server with Retell AI webhooks. The webhook endpoint receives events from Retell AI, processes the incoming JSON payload, and logs the event information.

## Prerequisites

- Python 3.10+
- FastAPI
- Uvicorn
- Retell Python SDK
- Postman (for local testing)
- ngrok (for exposing the local server)

Install the required packages:

```bash
pip install fastapi uvicorn retell-sdk
```

Set up environment variable

```bash
conda env config vars set RETELL_API_KEY="your_retell_api_key"
```
## Running the Server

Start the FastAPI application:

```bash
uvicorn voice_agent:app --reload
```

If testing locally, did that in Postman

Then push it online, expose the server using ngrok:

```bash
ngrok http http://127.0.0.1:8000
```

Copy the HTTPS URL generated by ngrok and configure it as the webhook URL in the Retell dashboard.

Example:

```
https://xxxxxxxx.ngrok-free.app/webhook
```

## Webhook Endpoint

The application exposes the following endpoint:

```
POST /webhook
```

## Testing Procedure

1. Start the FastAPI server.
2. Expose the local server using ngrok.
3. Configure the Retell webhook URL:

```
https://<ngrok-url>/webhook
```

4. Place a test call using Retell AI.
5. Observe the server console.

Expected output:

```
Call started: <call_id>
Call ended: <call_id>
Call analyzed: <call_id>
```

## Issues Encountered

### 1. 404 Not Found

**Problem**

Retell was sending requests to:

```
POST /
```

instead of:

```
POST /webhook
```

**Solution**

Update the webhook URL in the Retell dashboard to include the `/webhook` path.
